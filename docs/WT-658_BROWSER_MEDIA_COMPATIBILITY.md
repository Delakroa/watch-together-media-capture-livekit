# WT-658 — Browser media compatibility

## Цель

Расширить личный сценарий «открыть свой фильм и посмотреть вдвоём» без
нечестного обещания поддержки любого контейнера и без доставки неподтверждённого
FFmpeg в desktop installer.

## Поведение

- MP4/M4V H.264/AAC и WebM VP8/VP9/Opus остаются базовым, заранее заявленным
  путём. Для них выполняется browser MIME preflight, metadata и
  `captureStream()` проверка.
- File picker и drag-and-drop принимают другие видеофайлы, включая MOV, MKV,
  AVI, MPEG, TS, WMV и FLV. Они не получают разрешения на публикацию только
  по расширению.
- Для файла вне базовой политики host Chromium обязан реально загрузить
  metadata, декодировать первый кадр и отдать видеодорожку через
  `captureStream()`. Только после этого файл можно опубликовать в LiveKit.
- Успешный результат показывает `Экспериментально проверено на этом
устройстве` и предупреждение: это не базовая гарантия и зависит от decoder-а
  конкретного host-а.
- Ошибка metadata/decode/capture останавливает публикацию до передачи дорожек.
  Оригинальный фильм остаётся на компьютере host-а.

## Почему это не «любой файл»

Контейнер не сообщает, сможет ли Chromium декодировать его кодеки. Например,
MKV H.264/AAC может пройти на одном host-е, а HEVC, DTS или повреждённый файл
может быть отклонён. Это улучшает шанс открыть уже совместимый файл без
дополнительной установки, но не заменяет planned local normalizer из WT-651.

## Проверка

- unit-тест подтверждает, что MKV не блокируется пустым `canPlayType()` и
  получает experimental verdict только после успешного decode/capture.
- UI-тест закрепляет расширенный `accept` и честную пользовательскую подсказку.
- `pnpm check:ci` проверяет typecheck, frontend tests, lint, format и build.

## Следующий предел

Для стабильной поддержки HEVC, DTS и большинства старых/редких форматов нужен
отдельный desktop normalizer: platform encoder strategy, space/CPU UX,
auto-cleanup временных файлов, licenses/notices и реальные Mac/Windows
evidence. До такого решения S² не использует torrents, upload исходных файлов,
content catalog или DRM bypass.
